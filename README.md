# Telegram Dashboard MVP v5.8 — Real Contact Cards

إصدار v5.8 يضيف إلى v5.6 إرسال جهات الاتصال كبطاقات Telegram حقيقية، مع استمرار وحدة **المحتوى الإعلاني**:

- مكتبة محتوى محفوظة.
- محرر نص HTML مبسط لتنسيقات Telegram: غامق، مائل، تحته خط، مشطوب، كود، اقتباس، رابط، مشوش، وإيموجي.
- دعم محتوى نصي، صورة + نص، ملف + نص، ملصق كملف، جهة اتصال حقيقية كبطاقة Telegram.
- صفحة معاينة وإرسال تجربة إلى قناة.
- اختيار محتوى محفوظ داخل صفحة الحملة.
- النشر متعدد الحسابات يستخدم نفس المحتوى لكل مهمة حساب.

## التشغيل على Windows

```powershell
python -m venv .venv
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python run.py
```

## الترقية من v5.6

انسخ من نسختك الحالية إلى هذه النسخة:

```text
.env
instance/app.db
```

لا تغيّر `SESSION_ENCRYPTION_KEYS` إذا كنت تريد بقاء حسابات Telegram القديمة تعمل.

## ملاحظات

- الملفات المرفوعة تحفظ محلياً داخل `instance/uploads`.
- عند الانتقال إلى Railway أو VPS يفضل نقل الملفات إلى تخزين دائم أو Volume.
- جهة الاتصال تُرسل الآن كبطاقة Telegram حقيقية باستخدام `InputMediaContact`، وليست كنص عادي.
- استخدم رقم الهاتف بصيغة دولية مثل `+9665xxxxxxxx` حتى تظهر بطاقة الاتصال بشكل صحيح.
- أرسل اختباراً إلى قناة قبل تشغيل حملة واسعة.

## v5.9 — Channel Manager

إضافة إدارة قناة Telegram:

- صفحة إعدادات القناة.
- اختيار حساب النشر.
- حفظ روابط الموقع والتسجيل والتواصل.
- قوالب منشورات جاهزة: فرصة بحثية، فرصة مختصرة، فهرس، خدمة، تعليمي، تذكير، ترحيبي، أسئلة شائعة.
- إنشاء مسودات منشورات من القوالب.
- معاينة وتعديل نص المنشور.
- إرسال تجربة.
- نشر مباشر إلى القناة.
- حفظ `telegram_message_id` ورابط المنشور بعد النشر للقنوات العامة.
- تحديد منشور فهرس رئيسي وتحديثه تلقائياً بروابط المنشورات المنشورة.

ملاحظة: يجب أن يكون حساب النشر مشرفاً في القناة ولديه صلاحية النشر، وصلاحية تعديل الرسائل لتحديث منشور الفهرس.

# v6.6 — Per-Account Campaign Scheduling

هذا الإصدار يضيف جدولة نشر الحملات حسب كل حساب Telegram بشكل مستقل.

## الميزات

- اختيار الحسابات المشاركة في الحملة.
- لكل حساب يمكن اختيار:
  - نشر حالاً.
  - استخدام وقت الحملة العام.
  - وقت خاص لهذا الحساب.
- اختيار المنطقة الزمنية عند إنشاء الحملة، مثل Asia/Riyadh.
- إنشاء حملة مختلطة: بعض الحسابات تبدأ فوراً وبعضها ينتظر موعداً محدداً.
- عرض موعد النشر لكل حساب داخل تفاصيل الحملة.
- المهام المجدولة تعمل عبر `scheduler_worker.py`.

## التشغيل

شغل الموقع:

```powershell
python run.py
```

ولتشغيل المهام المجدولة تلقائياً:

```powershell
python scheduler_worker.py
```

## ملاحظات

- يجب أن يكون الحساب نشطاً ومزامناً مع المجموعات.
- وقت `datetime-local` لا يحتوي منطقة زمنية، لذلك يتم تفسيره حسب المنطقة الزمنية المختارة في صفحة الحملة ثم تخزينه كـ UTC داخلياً.
- خيار "نشر حالاً" يبدأ مباشرة بعد إنشاء الحملة.
- لا يتم إعادة إرسال الرسائل إلى الأهداف التي حالتها `sent` عند الاستئناف.
